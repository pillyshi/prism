# Prism 🔬
### Language-Defined Feature Space for Interpretable Text Analysis

Prism is a framework for discovering interpretable features from text collections using Large Language Models. Instead of relying on black-box embeddings, Prism defines a feature space in natural language — every feature is a human-readable hypothesis scored by an NLI model.

---

## Motivation

Traditional text representations (embeddings, BoW) are powerful but opaque. LLM prompting is flexible but offers no structured output. Prism bridges these worlds: it uses LLMs to *discover* and *define* a feature space in natural language, then scores each text against that space to produce interpretable feature vectors.

---

## Core Concepts

### Feature

The only building block in Prism. A feature is a declarative hypothesis about a single text, suitable for NLI scoring:

```
"This text expresses satisfaction with the product."
"This text mentions a specific defect or failure."
"This text indicates the author intends to repurchase."
```

Features are generated by an LLM from the text collection, then scored by an NLI model to produce a continuous entailment score in [0, 1].

### Feature Matrix

Each text is scored against all features using NLI:

| Text | expresses satisfaction | mentions defect | intends to repurchase | ... |
|------|----------------------|-----------------|----------------------|-----|
| doc1 | 0.92 | 0.05 | 0.87 | ... |
| doc2 | 0.08 | 0.91 | 0.03 | ... |

This matrix `X` of shape `(n_texts, n_features)` is the central data structure.

### Feature Dependencies

For each feature `i`, Prism fits a linear model predicting `X[:, i]` from all other features (`X[:, j≠i]`). A high R² indicates the feature is redundant — it can be well-explained by others. This reveals the dependency structure of the feature space without requiring any labels.

---

## Pipeline

```
generate_features(texts)           LLM → List[Feature]
        ↓
score(texts, features)             NLI → X (n_texts × n_features)
        ↓
FeatureSelector.fit(X, features)   Linear regression per feature → dependency report
FeatureSelector.transform(...)     Remove redundant features → X', features'
        ↓
  ┌─────────────────────────────────────────────────────────┐
  │  With labels y:                                         │
  │  Prism.fit(X', y, features') → FitResult               │
  │  (linear model coefficients — interpretable report)     │
  └─────────────────────────────────────────────────────────┘
        ↓
TextSynthesizer.fit(X', features') Gaussian distribution → save/load
TextSynthesizer.sample(n, llm)     Synthesize new texts
```

## Quickstart

```python
from prism import Prism, FeatureSelector, TextSynthesizer
from prism.llm import LLMClient
import numpy as np

llm = LLMClient("gpt-4o")
prism = Prism(llm=llm, nli_model="cross-encoder/nli-deberta-v3-large")

texts = [...]
y = np.array([...])  # labels (binary or continuous)

# 1. Generate features from the text collection
features = prism.generate_features(texts, n=20)

# 2. Score texts against features → feature matrix X
X = prism.score(texts, features)

# 3. (Optional) Assign human-readable names
named = prism.name_features(features)

# 4. Analyze feature dependencies and remove redundant features
selector = FeatureSelector(r2_threshold=0.9).fit(X, features)
for dep in selector.dependencies_:
    print(f"R²={dep.r2:.2f}  {dep.feature.hypothesis}")
X2, features2 = selector.transform(X, features)

# 5. Fit a linear model with y → interpretable report
result = prism.fit(X2, y, features2)
print(f"score ({result.scoring}): {result.score:.3f}")
for coef, f in sorted(zip(result.coef, result.features), key=lambda x: abs(x[0]), reverse=True):
    print(f"  {coef:+.3f}  {f.hypothesis}")

# 6. Synthesize new texts from the feature distribution
synthesizer = TextSynthesizer().fit(X2, features2)
synthesizer.save("synthesizer.json")

synthesizer = TextSynthesizer.load("synthesizer.json")
synthetic = synthesizer.sample(5, llm=llm)
```

### Local models (Ollama / llama.cpp)

Prism supports any [LangChain](https://python.langchain.com/)-compatible LLM, enabling fully local inference:

```python
from langchain_ollama import ChatOllama
from prism import Prism

# Use Ollama for feature generation; any LLM for synthesis
ollama = ChatOllama(model="llama3.2", format="json")
prism = Prism(llm=ollama, nli_model="cross-encoder/nli-deberta-v3-large")
```

Different LLMs can be used for generation and synthesis:

```python
from prism.llm import LLMClient

ollama_json = ChatOllama(model="llama3.2", format="json")
gpt4 = LLMClient("gpt-4o")

prism = Prism(llm=ollama_json)          # Ollama for feature generation
features = prism.generate_features(texts)
X = prism.score(texts, features)

synthesizer = TextSynthesizer().fit(X, features)
texts = synthesizer.sample(10, llm=gpt4)   # GPT-4 for synthesis
```

## Feature Dependency Analysis

`FeatureSelector` reveals the structure of the feature space without requiring any labels. For each feature `i`, it fits a Lasso model on `X[:, j≠i]` to predict `X[:, i]` and reports the R²:

```python
selector = FeatureSelector(r2_threshold=0.9).fit(X, features)

for dep in selector.dependencies_:
    print(f"R²={dep.r2:.2f}  {dep.feature.hypothesis}")
    for p, c in zip(dep.predictors, dep.coef):
        print(f"  {c:+.3f}  {p.hypothesis}")
```

- **R² near 0** — independent feature; carries unique information
- **R² near 1** — redundant; well-explained by other features in the set

`transform()` removes features whose R² exceeds the threshold:

```python
X2, features2 = selector.transform(X, features)
```

## Text Synthesis

`TextSynthesizer` fits a multivariate Gaussian on `X` and generates new texts by sampling feature vectors and prompting an LLM:

```python
synthesizer = TextSynthesizer().fit(X, features)
synthesizer.save("synthesizer.json")           # persist fitted distribution

synthesizer = TextSynthesizer.load("synthesizer.json")
texts = synthesizer.sample(
    n=10,
    llm=llm,
    n_levels=3,        # discretize: LOW / MED / HIGH
    threshold=0.3,     # only include features above this score
    rng=np.random.default_rng(42),
)
```

### Synthesis Evaluation

```python
from prism import evaluate_fit, evaluate_generation

# How well does the Gaussian capture the original distribution?
texts_synth, X_sampled = synthesizer.sample_with_vectors(20, llm=llm)
fit_eval = evaluate_fit(X, X_sampled)
print(f"Fit Wasserstein (mean): {fit_eval.wasserstein.mean():.3f}")

# How faithfully does the LLM follow the target feature vectors?
X_scored = prism.score(texts_synth, features)
gen_eval = evaluate_generation(X_sampled, X_scored)
print(f"Generation MAE (mean): {gen_eval.mae.mean():.3f}")
```

## Design Principles

- **Interpretable by construction** — every feature is a natural language hypothesis; every coefficient explains why
- **No axis hierarchy** — a single `Feature` type covers both coarse dimensions and fine-grained properties
- **Independent steps** — generate, score, analyze dependencies, fit, synthesize can each be run with different models
- **NLI-native** — hypotheses are scored continuously by NLI; no discrete yes/no conversion needed
